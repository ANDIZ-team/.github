# ADR-004: Chrome 프로필 분리 유예

- 작성일: 2026-04-22
- 작성자: 장수지, 박성훈
- 상태: accepted

## 배경

엔디즈 운영 중 개인 Google 계정과 공용 Google 계정(`contact.andiz@gmail.com`) 사이에 자동 로그인 충돌이 발생했다. Gmail을 열면 개인 계정으로 자동 로그인되어, 공용 계정으로 전환하려면 매번 로그아웃·재로그인이 필요한 불편.

일반적 해결책은 **Chrome 프로필을 업무용과 개인용으로 분리**하는 것이다. 그러나 엔디즈가 현재 사용하는 도구가 대부분 Google SSO 기반(Firebase, Jira, Notion, Figma, GitHub 로그인 일부) 이라는 특성을 고려해야 했다.

문제 맥락:
- 프로필 분리의 장점은 "서로 다른 인증 세션·쿠키·북마크의 독립"
- 지금은 모든 도구가 동일 Google 계정으로 수렴되므로, 프로필을 분리해도 실질 분리 효과가 작다
- 오히려 프로필 두 개 관리·동기화 설정 비용이 추가로 든다
- Apple Developer(Apple ID 자체 인증)처럼 Google 외 인증 체계가 본격 도입되면 상황이 달라진다

## 결정

**Chrome 프로필 분리는 유예한다.** 당장은 동일 Chrome 프로필에서 **Google 다중 계정 로그인** 방식을 사용한다.

- URL 패턴으로 계정 구분: `/u/0/` = 개인(`jddudios@gmail.com`), `/u/1/` = 공용(`contact.andiz@gmail.com`)
- 엔디즈 업무 북마크는 `/u/1/` 포함 URL로 저장하여 클릭 시 자동으로 공용 계정 컨텍스트에서 열리도록 한다
- Bitwarden 확장 로그인은 프로필 분리 여부와 무관하게 항상 개인 이메일(`jddudios@gmail.com`)로 유지 — ADR-003의 옵션 A 구조

프로필 분리를 도입할 트리거 조건을 미리 정의한다. 조건이 충족되면 그 시점에 Andiz 전용 Chrome 프로필을 생성(공용 Google 계정으로 동기화 on)한다.

## 근거

### 유예를 선택한 이유

- **지금 분리의 실익이 작다** — 모든 도구가 공용 Gmail로 수렴하므로 쿠키·세션 분리의 실질 효과가 낮다.
- **과잉 준비 경계** — 엔디즈는 매주 출시 모델이라 "오버엔지니어링"을 피하는 것이 비용 통제에 중요하다. 문제가 실제로 발생하는 시점에 대응이 합리적.
- **Google 다중 계정 로그인으로 충분** — 현재 불편(자동 로그인 충돌)의 실질 해결책은 다중 계정 로그인 + 북마크 URL 규약만으로 충분하다.

### 프로필 분리 트리거 조건

아래 조건 중 하나라도 충족되면 Andiz 전용 Chrome 프로필을 도입한다:

- **Apple Developer 본격 셋업 시점** — Apple ID가 공용 Gmail과 독립된 인증 체계로 추가되어 세션 분리 이익이 커진다.
- **멤버 3명 이상으로 확장** — 업무·개인 컨텍스트 구분이 개인 생산성에 더 크게 영향.
- **공용 카드를 Chrome에 저장할 필요가 생길 때** — 개인 쇼핑 자동완성과 섞이면 결제 실수 위험.

### 폐기한 대안

- **지금 즉시 프로필 분리 도입**: 이익 대비 관리 비용이 더 크다. 지금 해도 6개월 뒤 Apple Developer 도입 때 어차피 재구성.
- **시크릿 모드로 공용 계정 작업**: 매번 로그인·확장 설치가 번거롭고 Bitwarden 확장이 시크릿 모드에서 제한적이라 비효율.
- **별도 브라우저(Safari 등)로 엔디즈 전용**: 북마크·확장·도구 설정이 분산되어 도구 사용성 저하.

## 결과

### 얻는 것

- 추가 설정 비용 없이 현재 불편 즉시 해결.
- 프로필·동기화 관리 오버헤드 회피.
- 도입 시점을 Apple Developer 셋업과 묶을 수 있어 한 번의 환경 재구성으로 처리 가능.

### 감수하는 것

- 실수로 개인 계정 컨텍스트에서 엔디즈 자산을 만들거나 반대의 실수 가능성(`/u/0/` vs `/u/1/` 혼동).
- 개인·업무 북마크가 같은 Chrome 프로필에 섞임.
- 장기적으로는 분리 도입이 필요하므로 마이그레이션 비용은 미래로 연기될 뿐 사라지는 것은 아님.

### 재검토 조건

- 위 트리거 조건(Apple Developer 도입 / 멤버 3명 이상 / 공용 카드 Chrome 저장) 중 하나가 현실화되는 즉시 본 ADR을 `superseded by ADR-NNN`으로 전환하고 프로필 분리 도입 ADR 작성.
- 개인/공용 계정 혼동으로 인한 실무 사고(잘못된 계정으로 공유·저장 등)가 2회 이상 누적되면 트리거 조건 미충족 상태에서도 도입 검토.
